Use this guide to find the right repository when contributing to tscircuit. Most repos use GitHub issues and [Algora bounties](https://algora.io/tscircuit/bounties).

## Core platform

| Repository | Purpose |
| --- | --- |
| [tscircuit/tscircuit](https://github.com/tscircuit/tscircuit) | Main framework — create electronics with TypeScript and React |
| [tscircuit/core](https://github.com/tscircuit/core) | Core compiler from TypeScript to Circuit JSON |
| [tscircuit/circuit-json](https://github.com/tscircuit/circuit-json) | Low-level schematic, PCB, and manufacturing representation |
| [tscircuit/props](https://github.com/tscircuit/props) | Component prop definitions shared across packages |
| [tscircuit/cli](https://github.com/tscircuit/cli) | Local development server and CLI tooling |
| [tscircuit/tscircuit.com](https://github.com/tscircuit/tscircuit.com) | Website, playground, and online editor |
| [tscircuit/runframe](https://github.com/tscircuit/runframe) | Runs tscircuit code in a web worker and renders previews |
| [tscircuit/eval](https://github.com/tscircuit/eval) | Evaluate tscircuit code to Circuit JSON in browsers and Node |

## How to pick a repo

1. **UI or docs fixes** → `tscircuit.com`, `docs`, or `pcb-viewer`
2. **Language/compiler bugs** → `core`, `tscircuit`, or `eval`
3. **PCB routing** → `tscircuit-autorouter`, `dsn-converter`, or `copper-pour-solver`
4. **Schematic layout** → `schematic-trace-solver` or `schematic-symbols`
5. **Part data** → `jlcsearch`

Check open bounties at [algora.io/tscircuit/bounties](https://algora.io/tscircuit/bounties) before starting work.
